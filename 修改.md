我建议**不要继续 patch `expandIfFragmented()`**，而是把整个 Root Detection 做成一个 **Scoring Pipeline**。

你的现在代码已经有三层：

```text
SiteRule
    ↓
ARTICLE_SELECTORS
    ↓
detectArticleRoot()
```

其实只差最后一步：

> **Candidate Ranking（候选重新评分）**

这样以后 90% 的站点都不用再写 rule。

代码：

```ts
function chooseBestRoot(
    candidate:Element
){

    const list:Element[]=[];

    let p:Element|null=candidate;

    for(let i=0;i<3;i++){

        if(!p)break;

        list.push(p);

        p=p.parentElement;
    }

    let best=list[0];

    let bestScore=-999;

    for(const node of list){

        const result=

            scoreArticleContainer(node);

        console.log(

            node,

            result.score,

            result.reasons

        );

        if(result.score>bestScore){

            best=node;

            bestScore=result.score;
        }

    }

    return best;
}
```

# 第六部分：评分缓存（非常推荐）

因为：

```
candidate

parent

grandparent
```

以后很多地方都会算。

直接缓存：

```ts
const scoreCache=

new WeakMap<Element,RootScore>();
```

```ts
function score(

node:Element

){

    if(

scoreCache.has(node)

    ){

        return scoreCache.get(node)!;
    }

    const result=

        computeScore(node);

    scoreCache.set(

        node,

        result

    );

    return result;
}
```

避免反复扫描 DOM。

## 最终整体流程

```text
SiteRule
    │
    ▼
ARTICLE_SELECTORS
    │
    ▼
bestCandidate
    │
    ▼
refineArticleRoot()
    │
    ▼
expandWrappers()
    │
    ▼
CandidateRanking
        │
        ├── candidate        score=72
        ├── parent           score=81
        └── grandParent      score=94
    │
    ▼
chooseBestRoot()
    │
    ▼
TreeWalker
    │
    ▼
extractBlocks()
```

我比较推荐这个方案，因为它将你的内容检测从**基于规则（Rule-based）**演进为**基于启发式评分（Heuristic Scoring）**。随着评分因子的不断积累，像 Webflow、WordPress、Ghost、Medium、Hugo、Substack、OpenAI、Anthropic、Cloudflare 等绝大多数博客都能通过统一逻辑识别，而无需不断增加新的站点专用规则。最终，`SiteRule` 只需要处理真正特殊的少数站点，而不是成为维护成本越来越高的规则集合。
