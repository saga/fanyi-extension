# 翻译结果页：低优先级元素视觉弱化计划

## 1. 问题定义

当前 `/translate/<url>` 和 `/<id>` 返回的是**注入翻译后的完整 HTML**。原始页面的广告 banner、站点 footer、导航 header、侧边栏等都会原样显示，与正文翻译结果混在一起，分散阅读注意力。

Walker 阶段已经用 `SKIP_SET` / `SEMANTIC_SKIP_TAGS` 拒绝了这些元素的**翻译**，但它们仍然作为原始 DOM 的一部分显示在最终页面上。

**目标**：在翻译结果页中，对非正文元素进行视觉弱化（而非删除），让用户注意力集中在翻译后的正文上。

---

## 2. 核心方案：Walker 标记 + CSS 弱化 + hover 恢复

不删除 DOM，只通过 CSS 降低视觉权重。鼠标悬停时恢复，避免误杀有用内容。

```text
Walker (acceptNode 阶段)
  └─ 识别低优先级元素 → 添加 data-fanyi-low-priority="true"

Pipeline (CSS 注入阶段)
  └─ 追加 .fanyi-low-priority 样式：opacity 0.35，hover 恢复
```

为什么不在 Pipeline 阶段删除？
- 删除会破坏页面完整性（某些 footer 含版权信息、header 含文章标题导航）
- 删除后无法恢复，误杀代价高
- 视觉弱化可逆，用户需要时 hover 即可查看

---

## 3. 低优先级元素识别规则

`lib/translate/blockExtractor/rules.ts` 新增 `isLowPriorityElement(el)`：

```typescript
const LOW_PRIORITY_SELECTORS = [
  // 语义标签（含 heading 的 header 除外，因为可能是文章标题区）
  { tag: 'nav' },
  { tag: 'footer' },
  { tag: 'aside' },
  { tag: 'header', condition: (el) => el.querySelector('h1,h2,h3,h4,h5,h6') === null },

  // 广告 / 推广
  { classPattern: /\bad\b|banner|sponsor|promo|affiliate/i },
  { idPattern: /\bad\b|banner|sponsor|promo/i },

  // 社交分享 / 评论 / 相关推荐
  { classPattern: /share|social|comment|related|recommend|sidebar/i },

  // 站点级 chrome（非文章内容）
  { classPattern: /navbar|site-nav|global-nav|topbar|subscribe|newsletter|cookie|consent/i },
  { tag: 'form', classPattern: /subscribe|newsletter/i },
];
```

> **注意**：这些元素 Walker 已经会 REJECT（不翻译），现在只是额外加一个低优先级标记，不改变 walker 的现有行为。

---

## 4. Walker 层修改

在 `acceptWalkerNode` 的**硬性拒绝之后**、返回 `FILTER_REJECT` 之前，对低优先级元素打标记：

```typescript
// 1) 硬性拒绝条件
if (SKIP_SET.has(tag) || ...) {
  // ⭐ 新增：标记低优先级（即使拒绝翻译，也记录为低优先级元素）
  if (isLowPriorityElement(el)) {
    el.setAttribute('data-fanyi-low-priority', 'true');
  }
  cache.rejected.add(el);
  counters.rejected++;
  return FILTER_REJECT;
}

// 对 SEMANTIC_SKIP_TAGS（footer/nav/aside）也打标记
if (SEMANTIC_SKIP_TAGS.has(tag)) {
  el.setAttribute('data-fanyi-low-priority', 'true');
  cache.rejected.add(el);
  counters.skipped++;
  return FILTER_REJECT;
}
```

> 用 `setAttribute` 而非 `classList.add`，因为 `classList` 可能在某些环境中（linkedom）不支持多个 class 操作，`setAttribute` 更可靠。

## 4.5 弹窗 / Cookie Banner / Overlay：直接隐藏（不弱化）

Popup、Cookie consent banner、Subscribe 弹窗、Paywall 遮罩等**不是页面内容的一部分**，而是 overlay 层，会遮挡正文，必须直接移除。

### 识别规则（`isOverlayElement(el)`）

```typescript
const OVERLAY_PATTERNS = [
  // Cookie / GDPR / 隐私同意
  { classPattern: /\bcookie\b|consent|gdpr|privacy-banner|cookie-banner/i },
  { idPattern: /\bcookie\b|consent|gdpr|privacy/i },

  // 通用弹窗 / Modal / Overlay
  { classPattern: /\bpopup\b|\bmodal\b|overlay|dialog|backdrop|lightbox/i },
  { idPattern: /\bpopup\b|\bmodal\b|overlay|dialog/i },
  { role: 'dialog' },

  // Substack 系列
  { classPattern: /subscription-popup|paywall|subscribe-popup|newsletter-modal/i },

  // 固定定位的干扰性 banner（顶部/底部 fixed bar）
  { tag: 'div', styleCheck: (el) => {
      const pos = (el as HTMLElement).style?.position;
      return pos === 'fixed' || pos === 'sticky';
    }
  },
];
```

### 与低优先级的区别

| | 低优先级元素 | 弹窗/Overlay |
|--|-------------|-------------|
| 属性 | `data-fanyi-low-priority="true"` | `data-fanyi-remove="true"` |
| CSS | `opacity: 0.35` | `display: none !important` |
| hover 恢复 | ✅ 是 | ❌ 否 |
| 原因 | 是页面内容，只是不重要 | 不是页面内容，遮挡阅读 |

### Walker 实现

在 `acceptWalkerNode` 最前面（甚至早于 `SKIP_SET` 检查）识别 overlay：

```typescript
// 弹窗 / overlay：直接标记移除，后续 CSS 隐藏
if (isOverlayElement(el)) {
  el.setAttribute('data-fanyi-remove', 'true');
  // 不 return REJECT，因为 overlay 内部可能还有需要翻译的文本？
  // 实际上 overlay 遮挡层内的文本（如 "Accept all cookies"）不需要翻译
  // 所以这里可以直接 REJECT
  return FILTER_REJECT;
}
```

> 实际实现时，`isOverlayElement` 检查可以放在 `SKIP_SET` 之前，因为这些元素本来就不应该被翻译。

## 5. Pipeline / CSS 层修改

在 `pipeline.ts` 注入的 style 标签中追加：

```css
/* 低优先级元素视觉弱化 */
[data-fanyi-low-priority="true"] {
  opacity: 0.35;
  filter: grayscale(60%);
  transition: opacity 0.2s ease, filter 0.2s ease;
}
[data-fanyi-low-priority="true"]:hover {
  opacity: 1;
  filter: none;
}

/* 弹窗 / Cookie Banner / Overlay 直接隐藏 */
[data-fanyi-remove="true"] {
  display: none !important;
  visibility: hidden !important;
  pointer-events: none !important;
}
```

设计原则：
- **`opacity: 0.35`**：足够淡，不抢注意力，但仍可辨识
- **`grayscale(60%)`**：进一步降低视觉权重，彩色广告 banner 变灰后干扰性大幅下降
- **`transition`**：hover 时平滑恢复，体验自然
- **不修改 `display` / `max-height`**：不折叠、不隐藏，保留页面布局完整性
- **弹窗类用 `display: none !important`**：彻底移除，不占用 viewport 空间，不恢复

## 6. 可选增强（第二阶段）

### 6.1 Query Param 控制

`/translate/<url>?dim=none` — 关闭弱化（保留原样）
`/translate/<url>?dim=strict` — 更严格（opacity 0.2）

默认 `dim=normal`（opacity 0.35）。

### 6.2 完全隐藏模式（激进）

`/translate/<url>?hide=chrome` — 对低优先级元素设 `display: none`

风险较高，作为可选参数，不作为默认行为。

### 6.3 可折叠（折叠而非删除）

对超长的 footer / 评论列表，用 CSS 限制最大高度：

```css
[data-fanyi-low-priority="true"][data-fanyi-collapsible] {
  max-height: 120px;
  overflow: hidden;
  mask-image: linear-gradient(to bottom, black 70%, transparent 100%);
}
```

> 暂不实现，第一阶段以 opacity + grayscale 为主。

---

## 7. 边界情况与风险

| 场景 | 处理 |
|------|------|
| 文章标题在 `<header>` 内 | `header` 含 `h1-h6` 时**不**标记低优先级，保留原样 |
| 侧边栏含文章目录（TOC） | `aside` 默认标记低优先级，但 TOC 通常含 `<nav>` 语义，也被弱化。如果 TOC 有用，用户 hover 即可恢复 |
| 广告图片无文字 | walker 不翻译图片，但 `img` 在广告容器内，父元素被标记后整片区域都会弱化 |
| 暗黑模式 / 自定义主题 | `opacity` + `grayscale` 与颜色无关，任何主题下都有效 |
| 误杀正文容器 | 正文容器不会被判定为低优先级（条件明确排除了 article/main/section 等），风险极低 |

## 8. 实施步骤（建议顺序）

1. **`rules.ts`** — 新增 `isLowPriorityElement(el)` 函数
2. **`walker.ts`** — 在 `acceptWalkerNode` 的拒绝分支中，对低优先级元素 `setAttribute('data-fanyi-low-priority', 'true')`
3. **`pipeline.ts`** — CSS 注入追加 `[data-fanyi-low-priority="true"]` 样式
4. **测试** — 验证广告 banner / footer / nav 在翻译结果页中是否被正确弱化，hover 是否恢复；含 heading 的 header 是否不被弱化
